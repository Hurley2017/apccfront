Railway Reservation System

 The Project

This page contains the fundamentals and documentation supporting this project.

The UI design

We used traditional HTML, JS and CSS to implement the UI.

Frontend and Backend

We used Flask, Python to implement distinguished Servers for both Frontend API's and Backend API's.

The Database

We used NO-SQL(Document Store). And as a Distributed Database We chose MongoDB-Atlas

Asynchronus Javascript and XML (AJAX)

To process HTTP requests from frontend to backend we used AJAX, It helped API's talk directly to the scripts and viceversa. Although the misleading naming becuase it consists of XML, We did not use XML HTTP requests but instead of that we used JSON texts for flexibility.

The MongoDB database

At the beginning of the project We were thinking of traditional RDBMS. But soon later we were introduced to No-SQL in BigData(A DSE paper of our current semester). Then soon we ventured the distributed system and databases and we took interest in MongoDB because of it's flexibility and ease of access.

Why choose Python?

As begginners We always tried and played around with different programming languages. Among them We chose Python because of it's wide library and functionality. And going with Python made this project easy to manage and more flexible to work with.

Continuous Integration/Continuous Delivery or Deployment (CI/CD)

During the life span of the project development, We continuously integrated new features and improvements and We delivered it to cloud simultaneously. Doing so, Our web project was never offline handling jobs it was meant to handle yet always available to accept newer versions.

Libraries We used

For frontend

We used Bootstrap - The world's most popular front-end open source toolkit, featuring Sass variables and mixins, responsive grid system, extensive prebuilt components, and powerful JavaScript plugins.

For backend

Our backend services are powered by - "Pytz", "Flask", "Pymongo", "Bson", "Flask-CORS", "Smtplib" and "Heroku-CLI"

This project is deployed on Heroku (Heroku is a platform as a service that enables developers to build, run, and operate applications entirely in the cloud)
